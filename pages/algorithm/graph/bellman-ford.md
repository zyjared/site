# Bellman-Ford 算法

## 介绍

Bellman-Ford 算法是一种计算图中单源最短路径的算法，可以处理图中有负权边的情况。与 Dijkstra 算法不同，Bellman-Ford 算法通过多次松弛操作逐步更新各个节点的最短路径，最终计算出源节点到所有其他节点的最短路径。该算法也能够检测图中是否存在负权回路。

## 复杂度

|                | 复杂度   | 描述                            |
| -------------- | -------- | ------------------------------- |
| 最优时间复杂度 | $O(V E)$ | 每一条边进行松弛操作，重复V-1次 |
| 最坏时间复杂度 | $O(V E)$ | 在最坏情况下，遍历所有边和节点  |
| 平均时间复杂度 | $O(V E)$ | 遍历所有边和节点                |
| 空间复杂度     | $O(V)$   | 存储最短路径和节点信息          |

## 特点

### 优点

- 能够处理图中存在负权边的情况，适用范围广。
- 可以检测图中是否存在负权回路。
- 比 Dijkstra 算法更为通用，适合用于有负权边的图。

### 缺点

- 时间复杂度较高，对于大型图的计算较慢，尤其是边的数量较多时。
- 如果图中存在负权回路，无法得到正确结果。

## 应用场景

Bellman-Ford 算法常用于以下场景：

- 网络中最短路径计算，尤其是处理负权边的情况。
- 金融领域，计算最优的资金流动路径。
- 求解负权回路检测问题。
- 其他涉及图的最短路径计算，特别是存在负权边的场合。
