# Dijkstra 算法

## 介绍

Dijkstra 算法是一种用于计算图中两点之间最短路径的算法，广泛应用于计算无负权图的单源最短路径问题。该算法通过贪心策略逐步扩展已知最短路径的节点，最终找到源节点到所有其他节点的最短路径。

## 复杂度

|                | 复杂度            | 描述                                               |
| -------------- | ----------------- | -------------------------------------------------- |
| 最优时间复杂度 | $O(E + V \log V)$ | 使用优先队列的情况下（E为边的数量，V为节点的数量） |
| 最坏时间复杂度 | $O(E \log V)$     | 在最坏情况下，遍历所有边和节点                     |
| 平均时间复杂度 | $O(E + V \log V)$ | 遍历所有节点和边                                   |
| 空间复杂度     | $O(V + E)$        | 存储图结构和最短路径                               |

## 特点

### 优点

- 适用于无负权图的单源最短路径问题，效率高。
- 通过使用优先队列（堆），能够在每一步选择最短的节点进行处理，从而保证每次选取最优解。

### 缺点

- 不适用于有负权边的图，不能处理负权边的情况。
- 在图较大时，空间复杂度较高。

## 应用场景

Dijkstra 算法广泛应用于以下场景：

- 路径规划问题，如导航系统。
- 网络中最短路径问题。
- 任务调度问题，尤其是在需要优化时间和资源的情况下。
- 其他涉及图的最短路径计算的应用。
