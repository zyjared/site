# Prim 算法

## 介绍

Prim 算法是另一种求解最小生成树 (MST) 的贪心算法。与 Kruskal 算法不同，Prim 算法从一个节点开始，逐渐扩展最小生成树，选取当前生成树外的边中权重最小的边，将相应节点加入生成树中，直到所有节点都被加入为止。该算法常常使用优先队列（堆）来实现高效的边的选择。

## 复杂度

|                | 复杂度        | 描述                                           |
| -------------- | ------------- | ---------------------------------------------- |
| 最优时间复杂度 | $O(E \log V)$ | 由于优先队列的操作，时间复杂度为 $O(E \log V)$ |
| 最坏时间复杂度 | $O(E \log V)$ | 边的处理和优先队列的操作                       |
| 平均时间复杂度 | $O(E \log V)$ | 时间复杂度对边数和节点数的关系                 |
| 空间复杂度     | $O(V + E)$    | 存储节点、边以及优先队列的空间                 |

## 特点

### 优点

- 算法可以高效地构造最小生成树，适用于稠密图。
- 适用于边较多的情况，时间复杂度相对较低。
- 使用优先队列（堆）优化了边的选择过程，使得算法效率更高。

### 缺点

- 如果使用不当，可能需要更多的内存（比如存储图的邻接矩阵或邻接表）。
- 需要额外的优先队列结构来优化查找最小边的过程。

## 应用场景

Prim 算法广泛应用于以下场景：

- 网络设计：如构建最低成本的网络。
- 图的优化问题：尤其是当图比较稠密时，Prim 算法表现出色。
- 适用于构建电网、道路网等连接问题中最小生成树。
