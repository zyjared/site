# 二分查找

## 介绍

二分查找是一种高效的查找算法，要求数据是有序的。其基本思想是通过将数组分成两半来逐步缩小查找范围。每次比较中间元素与目标元素的大小，决定继续在左半部分或右半部分查找。

## 复杂度

|                | 复杂度      | 描述                   |
| -------------- | ----------- | ---------------------- |
| 最优时间复杂度 | $O(1)$      | 目标元素正好是中间元素 |
| 最坏时间复杂度 | $O(\log n)$ | 每次将搜索区间减少一半 |
| 平均时间复杂度 | $O(\log n)$ | 每次将搜索区间减少一半 |
| 空间复杂度     | $O(1)$      | 只需要常数级空间       |

## 特点

### 优点

- 时间复杂度为 $O(\log n)$，比线性查找要高效得多，特别是数据量大的时候。
- 只需要常数级别的空间。

### 缺点

- 仅适用于已排序的数据。
- 插入和删除操作的时间复杂度较高，需要先对数据进行排序。

## 应用场景

二分查找广泛应用于需要频繁查询的场景，如查找表、数据库索引、搜索引擎等，尤其是在有序数据集中的查找任务中表现突出。
