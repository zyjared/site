# 广度优先搜索

## 介绍

广度优先搜索（BFS）是一种用于遍历或搜索树或图的算法。与深度优先搜索不同，广度优先搜索从根节点开始，沿着树的宽度遍历，首先访问所有的邻居节点，然后再遍历这些邻居节点的邻居。BFS 使用队列来实现，确保按层次顺序遍历图中的节点。

## 复杂度

|                | 复杂度     | 描述                   |
| -------------- | ---------- | ---------------------- |
| 最优时间复杂度 | $O(V + E)$ | 遍历所有节点和边       |
| 最坏时间复杂度 | $O(V + E)$ | 遍历所有节点和边       |
| 平均时间复杂度 | $O(V + E)$ | 遍历所有节点和边       |
| 空间复杂度     | $O(V)$     | 存储已访问的节点和队列 |

## 特点

### 优点

- 适用于求解最短路径问题，尤其是在无权图中。
- 通过层次遍历，能更快地找到目标节点。
- 适用于广泛的图结构，尤其是在图较为稀疏时。

### 缺点

- 相比于深度优先搜索，空间复杂度较高，因为需要存储所有已访问的节点和待访问节点。
- 如果图很大，可能会导致内存消耗较大。

## 应用场景

广度优先搜索广泛应用于以下场景：

- 求解无权图的最短路径问题。
- 图的层次遍历。
- 在社交网络分析中查找最短路径。
- 在人工智能中，用于路径寻找、迷宫问题等。
