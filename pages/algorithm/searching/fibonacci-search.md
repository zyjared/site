# 斐波那契查找

## 介绍

斐波那契查找是一种基于斐波那契数列的查找算法，适用于有序数组。与二分查找类似，斐波那契查找通过缩小查找范围来定位目标元素，但不同的是它通过斐波那契数列来确定分割点，而不是直接取中间位置。

## 复杂度

|                | 复杂度      | 描述                                          |
| -------------- | ----------- | --------------------------------------------- |
| 最优时间复杂度 | $O(1)$      | 目标元素正好是分割点元素                      |
| 最坏时间复杂度 | $O(\log n)$ | 每次缩小查找范围，最多进行 $O(\log n)$ 次比较 |
| 平均时间复杂度 | $O(\log n)$ | 每次分割能够有效减少查找范围                  |
| 空间复杂度     | $O(1)$      | 只需要常数级空间                              |

## 特点

### 优点

- 在某些情况下，斐波那契查找可能比二分查找更有效，尤其是在数组元素较多时。
- 时间复杂度为 $O(\log n)$，适用于大规模的有序数据集。

### 缺点

- 仅适用于有序数组。
- 相比于二分查找，斐波那契查找的实现更为复杂。

## 应用场景

斐波那契查找适用于有序数据集，特别是当数组大小不确定或需要处理分段数据时。可以用于数据库索引、文件查找等场景。
