# 线性查找

## 介绍

线性查找是一种简单的查找算法，基本思想是从数组的第一个元素开始，依次与目标元素比较，直到找到目标元素或遍历完整个数组。它不要求数据必须是有序的，适用于小规模数据或数据不确定的情况。

## 复杂度

|                | 复杂度 | 描述                                             |
| -------------- | ------ | ------------------------------------------------ |
| 最优时间复杂度 | $O(1)$ | 目标元素在数组的第一个位置                       |
| 最坏时间复杂度 | $O(n)$ | 目标元素在数组的最后位置，或数组中不存在目标元素 |
| 平均时间复杂度 | $O(n)$ | 每个元素都有可能是目标元素                       |
| 空间复杂度     | $O(1)$ | 只需要常数级的空间                               |

## 特点

### 优点

- 实现简单，不需要额外空间。
- 不要求数据有序，适用于任何类型的数组。

### 缺点

- 在最坏情况下，需要遍历整个数组，时间复杂度较高。
- 对于大规模数据，性能较差。

## 应用场景

线性查找适用于小规模或无序数据的查找，特别是当数组大小不大时，线性查找非常实用。它常用于简单的应用场景，如查找元素是否存在、查找匹配项等。
