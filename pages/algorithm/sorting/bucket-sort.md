# 桶排序

## 介绍

桶排序（Bucket Sort）是一种基于分布的排序算法，它将数据分散到多个桶中，然后对每个桶内的元素进行排序，最后再将所有桶中的元素合并。桶排序的核心思想是将数据按照一定的规则分配到不同的桶中，然后对每个桶内部的元素进行排序。桶排序特别适用于数据均匀分布的情况。

## 复杂度

|                | 复杂度     | 描述                                                                     |
| -------------- | ---------- | ------------------------------------------------------------------------ |
| 最优时间复杂度 | $O(n)$     |                                                                          |
| 最坏时间复杂度 | $O(n^2)$   | 当数据集中在某些桶时，每个桶可能包含所有元素，需要对其进行 $O(n)$ 的排序 |
| 平均时间复杂度 | $O(n + k)$ | 平均情况下，时间复杂度为 $O(n + k)$，$n$ 是元素数量，$k$ 是桶的数量      |
| 空间复杂度     | $O(n + k)$ | 需要额外空间存储桶和桶内的元素                                           |

## 特点

### 优点

- 对于数据均匀分布的情况，桶排序具有很好的性能，时间复杂度接近 $O(n)$。
- 可以实现并行化，因为每个桶内的排序是独立的，可以并行进行。
- 算法稳定，保证相等元素的相对顺序。

### 缺点

- 适用数据集要求较高，需要数据分布均匀。
- 对于数据集中度较高的情况，桶排序的效率会降低，因为桶内的元素排序会导致较高的时间复杂度。
- 空间复杂度较高，需要额外的存储空间。

## 应用场景

桶排序适用于数据分布均匀的场景，尤其是在数字范围有限且分布均匀的情况下，如浮点数排序。它还适用于外部排序问题，可以通过将数据分配到不同的桶中来提高排序效率。
