# 堆排序

## 介绍

堆排序（Heap Sort）是一种基于比较的排序算法，它利用堆这一数据结构来实现排序。堆是一种完全二叉树，可以通过堆的性质（父节点大于子节点）来进行排序。堆排序分为两个阶段：首先将数据构建成一个大顶堆，然后反复将堆顶元素（最大元素）与最后一个元素交换，并重新调整堆。

堆排序是一种不稳定排序算法。

## 复杂度

|                | 复杂度         | 描述                                                     |
| -------------- | -------------- | -------------------------------------------------------- |
| 最优时间复杂度 | $O(n \log n)$  | 无论数据是否有序，堆排序的时间复杂度始终为 $O(n \log n)$ |
| 最坏时间复杂度 | $O(n \log n)$  | 堆排序始终为 $O(n \log n)$，不受数据顺序影响             |
| 平均时间复杂度 | $O(n \log n)$  | 时间复杂度始终是 $O(n \log n)$                           |
| 空间复杂度     | $O(1)$，$O(n)$ | 原地排序，不需要额外空间                                 |

## 特点

### 优点

- 堆排序的时间复杂度为 $O(n \log n)$，性能较为稳定。
- 原地排序，不需要额外的存储空间，空间复杂度为 $O(1)$。
- 不受输入数据的顺序影响，堆排序总是表现出一致的性能。

### 缺点

- 堆排序是一个不稳定的排序算法，相等元素的相对顺序可能会改变。
- 在实际应用中，堆排序的常数因子较大，性能不一定优于快速排序和归并排序。

## 应用场景

堆排序适用于对内存使用要求较高的情况，因为它是原地排序，且空间复杂度较低。常用于处理优先队列等问题，也适用于大规模数据排序，尤其是数据规模较大的情况。
