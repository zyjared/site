# 基数排序

## 介绍

基数排序是一种非比较型的整数排序算法，它通过将数字按位数分配到不同的桶中，然后逐位排序，最终得到有序的结果。基数排序是通过对数据进行多轮排序来实现的，每一轮排序按数字的每一位进行处理，从最低位到最高位或从最高位到最低位。基数排序适用于整数排序，也可以用于固定长度的字符串排序。

## 复杂度

|                | 复杂度     | 描述                                                     |
| -------------- | ---------- | -------------------------------------------------------- |
| 最优时间复杂度 | $O(nk)$    | $k$ 是位数，$n$ 是元素数量，最优情况为数字较小，位数较少 |
| 最坏时间复杂度 | $O(nk)$    | $k$ 是位数，$n$ 是元素数量，在大规模数据时，$k$ 可能较大 |
| 平均时间复杂度 | $O(nk)$    | 平均情况下，$k$ 是位数，$n$ 是元素数量                   |
| 空间复杂度     | $O(n + k)$ | 需要额外的存储空间，用于计数和排序过程                   |

## 特点

### 优点

- 基数排序时间复杂度与数据的位数和数量有关，适用于大范围整数数据。
- 是稳定排序算法，不会改变相等元素的相对顺序。
- 可以高效地处理大范围的数字。

### 缺点

- 适用于数字排序，且只能处理整数或有固定长度的字符串。
- 空间复杂度较高，需要额外的存储空间来存放计数数组和输出数组。
- 需要对数据的位数有一定的了解，位数过多会导致性能问题。

## 应用场景

基数排序适用于对整数或有限范围的数字进行排序，特别是在数据范围较大且数据量也很大时。它在处理大量整数时，效率较其他比较排序算法高，常用于排序大量的无符号整数或字符串（如IP地址、电话号码等）。
