# 选择排序

## 介绍

选择排序（Selection Sort）是一种简单的排序算法，基本思想是每次从未排序的部分选出最小（或最大）元素，放到已排序部分的末尾。该算法通过不断交换最小值与当前未排序部分的第一个元素来逐步排序。

## 复杂度

|                | 复杂度 | 描述                                               |
| -------------- | ------ | -------------------------------------------------- |
| 最优时间复杂度 | O(n²)  | 无论输入数据是否有序，算法都需要进行相同次数的比较 |
| 最坏时间复杂度 | O(n²)  | 每次都需要进行最小值交换                           |
| 平均时间复杂度 | O(n²)  | 每个元素大致需要 $n$ 次比较                        |
| 空间复杂度     | O(1)   | 原地排序，不需要额外空间                           |

## 特点

### 优点

- 实现简单，适用于小规模数据集。
- 稳定排序，适合用于已排序或接近排序的数据。

### 缺点

- 时间复杂度较高，特别是在大规模数据集上，效率较低。
- 选择排序不稳定，若存在重复元素，排序后可能改变其相对顺序。

## 应用场景

选择排序适用于小规模数据排序，尤其是当数据集已经接近有序时，它的性能会更好。选择排序也可用于内存空间受限的场景，因为它是原地排序，空间复杂度为 $O(1)$。
